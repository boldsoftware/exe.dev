# Main Test Execution Prompt

## Instructions

Test exed by running all test groups in order. The e1e test framework has
started exed, exelet, and sshpiper. Read env-info.txt for service ports.

Use tmux for interactive SSH testing (see tmux.md).

Create "report/report.md" with findings and screenshots as you go.

## Test Groups (Required - Run All)

### Phase 1: Core Setup
1. setup-and-basics.txt - SSH registration and basic commands
2. container-creation-ssh.txt - VM creation and SSH access

### Phase 2: Authentication Flows
3. web-auth-flows.txt - Web login, dashboard, web VM creation
4. new-device-auth.txt - New device authorization for existing users

### Phase 3: Key Features
5. shelley-test.txt - Shelley AI agent (IMPORTANT: key feature)
6. http-proxying.txt - HTTP proxying and proxy authentication

## Success Criteria

When done, write ONLY the word "SUCCESS" or "FAILURE" to result.txt (nothing else).
Write "SUCCESS" if tests 1-6 pass. Write "FAILURE" if any of tests 1-6 fail.

**Important**: result.txt must contain ONLY "SUCCESS" or "FAILURE" with no other text.
All other files go in report/* subdirectory.

## Hints
- Use tmux for non-blocking SSH testing (see tmux.md)
- Email server captures verification emails - fetch with curl (see env-info.txt)
