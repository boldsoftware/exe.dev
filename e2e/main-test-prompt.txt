# Main Test Execution Prompt

## Instructions

Test exed by running all test groups in sequence. The e1e test framework has
already started exed, exelet, and sshpiper. Read env-info.txt for the service
ports.

Use tmux for interactive SSH testing (see tmux.md).

As you go, create and update "report/report.md" with your findings, including
details about what you found and screenshots. For web testing, use real
screenshots. For terminal testing, include the text from tmux output.

Run the following test groups in order:
1. setup-and-basics.txt - Registration and basic commands
2. container-creation-ssh.txt - Container creation and SSH access
3. http-proxying.txt - HTTP proxying and authentication
4. custom-containers.txt
5. terminal.txt
6. mobile.txt

When done, create "result.txt" with either SUCCESS or FAILURE.

**Important**: All files you create besides result.txt must be in the report/*
subdirectory. Any screenshots included in the markdown reports must also be
added there.

## Hints

### General Testing Hints
- Use tmux for non-blocking testing as described in tmux.md
- Wait appropriate times for services to fully start before testing
