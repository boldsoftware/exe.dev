# Setup and Basic Operations Test

## Instructions

Test basic exed setup, registration, and fundamental commands.

### Test Setup

1. Verify that `ssh colima-exe-ctr echo hello` works before you start. (The VM runs containerd + kata + cloud hyprvisor + nydus and should be running.)

2. Build and start exed (in dev mode).

3. Build and start sshpiper.

4. Create yourself an ssh key with ssh-keygen.

### Test Steps

#### Registration

1. Register with exed by running "ssh -p 2222 localhost" 

The e-mail thing will require doing a web call; use the browser to handle that.

2. Test the "whoami" command.

3. Test the "list" command (should have no machines yet!)

## Hints

### Key Discovery
- exed runs on port 8080 (HTTP) and 2223 (SSH direct)
- sshpiper runs on port 2222 (SSH proxy) 
- Email verification requires browser interaction at http://localhost:8080/verify-email?token=...
- SSH to containers: `ssh -p 2222 machinename@localhost` (wait ~15s after creation)
- Build times: exed ~30 seconds first time, sshpiper ~10 seconds
- Check logs: `tmux capture-pane -p -t testing:exed` or `testing:sshpiper`

### Setup Hints
- Use tmux windows: `tmux new-session -d -s testing && tmux new-window -t testing -n exed && tmux new-window -t testing -n sshpiper && tmux new-window -t testing -n client`
- We cannot directly run an exe-ctr host because it requires /dev/kvm, that is why we work through exe-ctr-colima.
- Build: `cd /app && make build`. Start: `cd /app && make run-dev` (in exed window). Server starts on :8080 (HTTP) and :2223 (SSH). Look for "SSH server starting" in output.
- Build: `(cd deps/sshpiper && go build -o sshpiperd ./cmd/sshpiperd && go build -o metrics ./plugin/metrics)`. Start: `./sshpiper.sh` (in sshpiper window). Listens on :2222.
- Command: `ssh-keygen -t rsa -b 2048 -f ~/.ssh/id_rsa -N ""`. Creates RSA key needed for authentication.

### Registration Hints
- Use: `ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null -p 2222 localhost`. Prompts for email and team name.
- After entering email/team, get verification URL like http://localhost:8080/verify-email?token=... Copy this URL to browser and click "Confirm Email Verification"
- After email verification, press any key to continue, then type `whoami` at exe.dev prompt. Shows email and SSH key fingerprint.
- Type `list` at exe.dev prompt. Should show "No machines found. Create one with 'new'."
