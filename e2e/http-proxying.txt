# HTTP Proxying Test

## Instructions

Test HTTP proxying capabilities, authentication flows, and proxy management.

### Prerequisites

- Registration must be completed (setup-and-basics.txt)
- Container must be created and SSH access working (container-creation-ssh.txt)
- Read env-info.txt for service ports

### Test Steps

#### HTTP Proxying

1. Start an HTTP server on your VM (e.g., python3 -m http.server 8000)

2. Connect to it via http://vmname.exe.cloud:<HTTP_PORT>/. There should be a login flow.

Note that there should be a confirmation interstitial screen for logging into
the VM using your exe.dev credentials.

Note also that if you were logged out to begin with, you should end up at the exact
URL you started at.

3. Logout on http://localhost:<HTTP_PORT>/

4. Connect again; you should require login; don't log in.

5. Use the "share" command in the exe.dev shell to make the proxy public. You should be able to access it without auth.

6. Use http://vmname.exe.cloud:<HTTP_PORT>/__exe.dev/logout to test logging out of a specific subhost.


## Hints

### HTTP Server Hints
- In the container, start a simple HTTP server: `python3 -m http.server 8000`
- Access via browser at http://vmname.exe.cloud:<HTTP_PORT>/
- The system should redirect through authentication flows

### Authentication Hints
- Login flow may involve redirections and session management
- After logout, accessing container URLs should require re-authentication

### Route Management Hints
- Use `share` command in exe.dev shell to manage public/private access
- Making routes public bypasses authentication requirements
- Test both authenticated and unauthenticated access patterns
