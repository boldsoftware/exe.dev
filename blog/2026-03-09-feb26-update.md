February kept us busy! Thanks to all our users for the great feedback, so many bug fixes and quick improvements were possible because of high quality reports.

Here are a few things we shipped in February:

## Idea templates

The [new VM page](https://exe.dev/new) now has a gallery of **idea templates**: one-click setups for apps like Gitea, VS Code, Ghost, Minecraft, Grafana, Outline, and more. Each template comes with a tailored prompt that Shelley uses to set up the app automatically, including auth configuration. And of course it includes the most popular one, OpenClaw.

More than anything, the goal of idea templates is to demonstrate what you can do with a prompt. That empty text box is far more powerful than it looks.

Take a look at them at [exe.dev/idea](https://exe.dev/idea).

## HTTPS API

We published an HTTPS [API](https://exe.dev/docs/https-api). You can now script exe.dev programmatically — create VMs, manage them, and integrate with your existing tooling — from systems that cannot SSH. There is a particularly fun approach to API tokens using SSH key signing.

## Shelley

Shelley saw a lot of work this month:

- **Conversation distillation.** When a conversation gets long, Shelley can now distill it — using an LLM to summarize the context — and continue in a fresh conversation without losing the thread.

- **Browser tools** got a major overhaul. Seven separate tools were consolidated into a single `browser` tool with sub-actions for navigation, screenshots, JavaScript evaluation, and new capabilities: device emulation, network monitoring, accessibility tree inspection, and performance profiling.

- **Notifications.** Shelley can now notify you when a long task finishes — via browser notification, Discord, or email.

- **Internationalization.** The UI is now available in 5 languages, with a language dropdown and per-language commands in the ⌘K palette.

- **Terminal panel.** A built-in terminal panel sits below the conversation, with proper dark mode, persistent sessions across conversations, and tab support.

- **`llm_one_shot` tool** for quick, stateless LLM calls — summaries, classification, reformatting — without spinning up a subagent.

- **Markdown toggle.** A three-mode toggle (off / agent-only / all) controls markdown rendering in the conversation view.

- **Archive conversations** from the overflow menu and browse them later.

- **Model lock.** Each conversation now locks to the model it started with, so switching your default model doesn't retroactively change ongoing conversations. The model name is visible in the context usage popup.

- **Performance.** Fixed O(n²) memory growth in long conversations, added LRU caching for conversation messages, and memoized rendering for large conversations.

- New models: **Claude Opus 4.6**, **Claude Sonnet 4.6**, **GPT-5.4**, and **GPT-OSS-20b**.
