Observation Log: Jeff Onboarding to the PaaS (“Just use SSH”)

Jeff begins the onboarding with curiosity and mild skepticism.

First impressions:
“Just use SSH? I’m down, but still not sure what this is.”
“Feels old school… sounds pretty cool.”

He opens the docs, scans the FAQ, and heads to the dashboard before realizing he’s unsure about the flow.
“Is there user creation? Is that what this is?”
“Email loopback? What do I need to do?”

He proceeds with the “SSHish in” flow — enjoys the animation — and notes:
“On interactive signup.”

The terminal freezes midway. He opens a new session and it works. Completes signup (no GitHub SSH key present), but it freezes again in Terminal.app. Eventually he reaches the menu.

He types help, and asks:
“What is Box routing?”
“What is web authenticated?”

Jeff moves to the UI, clicks +New, chooses Shell, and runs ls:
“no boxes.”

Back to terminal to try ssh command there. It works and he is in. He reads the banner and says:

“There is no AI! So happy! — Oh.  But there is. :( — Not in your face, so cool.” (Started this before getting to footnote on Shelly.)

He opens Shelly, starts a new conversation:
“What is this?”

⸻

Exploring the Product

Back in the menu, Jeff runs ls again, then attempts to SSH:
ssh thing@exe.dev → “no ssh command” (he’s still in the menu).
Opens a new terminal, SSHs directly to the box — success.

He clicks the HTTPS link in the UI for his box and looks puzzled:
“Auth?”
“How do you pronounce exe.dev?”
“My VSCode? Why not in the browser??”

A system prompt warns about opening an external app. He installs the necessary plugins and setup tools, having never done it before.
“I use Cursor,” he says when asked about VSCode.
“I don’t use the ssh tunneling stuff in these IDE things.”

He’s visibly confused.
“I thought this would be a dev environment.”

Tries the VSCode link again:
“You could eliminate this whole thing by having it run in the browser.”

Back in the terminal, ls shows the “welcome app” and SQLite files.
In the UI, he hovers over the SSH button, clicks it — gets “permission denied” (wrong SSH key in VSCode terminal).

Back in Shelly:
“What files are in the app directory?”
Shelly shows ls output.
“What is welcome?” -- Shelly can't say for sure but attempts to infer from contents in dir.

He notices hover debug metrics: “Confused.”
Runs cd /app/welcome && make build from Shelly; spinner appears.

“Overall, this is very cool,” he says.
Jeff is drawn to Shelly, seemingly thinking it is the product.
“Open proxy means public URL?”

⸻

Digging Deeper (System Exploration)

He SSHs into the box, navigates /etc, then /etc/sysctl.d.
Confused that /etc isn’t visible in VSCode’s file tree, though it appears in the terminal.

Runs make build.
Error: polkit.service is masked.

Tries make stop.
“Can Shelly do it?”
Switches back to Shelly, which attempts kill(1) and fails.

He checks systemctl status, sees both “welcomed” and “shelly.”
“I think I killed the example app. Maybe?”

Checks port 8000: nothing running.
“Can I sudo??” Tries and does.

He’s not copying command output between Shelly and SSH sessions.
curl localhost:8000 — no connection.
Runs ifconfig.
Systemd still shows “welcomed” as active.

We're nearly out of time, so I stop him and give him a demo of ephermeral VMs and how --image can launch docker containers, and how "proxy" works (which he was super confused by until I showed him).

After my 7 minute demo, I asked for feedback. Below are my quick notes as he feed his thoughts to me:

⸻

Jeff’s Summary Impressions

“It’s multi-modal — in that I have SSH and containers.”
“It’s a nice re-packaging of things already there.”
“I’m excited about non-HTTP ports.”
“I just want to run a server.”
“I can also just use ngrok for TCP.”
“It has a cool factor that I like.”
“I’d be excited to see it become more of a toolkit — show all the different ways I can use the privates it provides.”
“‘Everything’ is quick and easy VMs (like DigitalOcean). ‘Fast VM!’ isn’t a hook for me.”
“Felt disjointed — but once I figured it out… Is there an API? Can I use this as generic compute?”
“I want to think of it as a tool for making my own platform.”
“It’s been a long time since I saw a cool developer product like this.”
