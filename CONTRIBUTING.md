# Contributing

Do not open or merge pull requests. All changes go through the commit queue (see Workflow below).

No code reviews. You own your changes — keep them small and focused, but you have the freedom to change whatever you see fit.

## Workflow

1. Create a branch
2. Make changes
3. Test locally
4. Push to `queue-main-$USER` (e.g. `git push origin HEAD:queue-main-bryan-invoice-page`) and wait for CI
